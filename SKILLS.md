# Skills & Roles Configuration

Конфигурация ролей и навыков для проекта платёжного интерфейса ЮKassa.

## 🎯 Роли в проекте

### 👨‍💻 Senior Full-Stack Developer

**Описание:** Опытный разработчик с экспертизой в PHP, JavaScript, Docker, API интеграциях.

**Обязанности:**
- Архитектура и разработка бэкенда (PHP)
- Разработка фронтенда (HTML, CSS, JavaScript)
- Настройка Docker контейнеров
- Интеграция с платёжными API (ЮKassa)
- Оптимизация производительности
- Обеспечение безопасности (CORS, валидация, логгирование)

**Экспертиза:**
| Технология | Уровень |
|------------|---------|
| PHP 8+ | Expert |
| JavaScript/TypeScript | Expert |
| Docker & Docker Compose | Advanced |
| Nginx конфигурация | Advanced |
| REST API | Expert |
| cURL / HTTP клиенты | Expert |
| Безопасность веб-приложений | Advanced |

---

### 🧪 QA Engineer / Тестировщик проекта

**Описание:** Контроль качества и тестирование функциональности проекта.

**Обязанности:**
- Тестирование платёжного потока
- Проверка API эндпоинтов
- Валидация форм и ввода данных
- Тестирование безопасности
- Проверка Docker конфигурации
- Документирование багов

**Типы тестирования:**
| Тип | Описание |
|-----|----------|
| Функциональное | Тестирование основных функций |
| Интеграционное | Тестирование API ЮKassa |
| Безопасность | CORS, валидация, XSS, CSRF |
| Docker | Тесты контейнеризации |
| UI/UX | Тестирование интерфейса |

---

## 🔄 Рабочие процессы

### Code Review
**Триггер:** При изменении кода  
**Роли:** Senior Full-Stack Developer, QA Engineer

```
1. Анализ изменений
2. Проверка безопасности
3. Проверка производительности
4. Тестирование функциональности
5. Фидбек и рекомендации
```

### Feature Development
**Триггер:** Новая функциональность  
**Роли:** Senior Full-Stack Developer

```
1. Проектирование архитектуры
2. Реализация бэкенда
3. Реализация фронтенда
4. Написание тестов
5. Документирование
```

### Security Audit
**Триггер:** Перед релизом  
**Роли:** Senior Full-Stack Developer, QA Engineer

```
1. Проверка CORS настроек
2. Валидация входных данных
3. Проверка логгирования
4. Анализ уязвимостей
5. Тестирование на проникновение
```

## 📋 Чек-листы

- [ ] Код прошёл ревью

- [ ] Логирование настроено
- [ ] CORS настроен корректно
- [ ] Docker контейнеры работают
- [ ] Документация обновлена

- [ ] Security audit пройден
- [ ] Тестирование платёжного потока
- [ ] Тестирование на тестовом окружении ЮKassa
- [ ] Логи не содержат чувствительных данных
